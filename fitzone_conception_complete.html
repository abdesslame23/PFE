Ce diagramme structure les entités du système de réservation et de pointage, définissant leurs propriétés et associations.

Détail des associations :

  • User - Abonnement : Un membre possède une liste historique de contrats d'abonnement (1 à plusieurs).

1.3 — Diagramme de Séquence (Pointage par Code)

Modélisation dynamique du processus de pointage physique aux portiques d'entrée de la salle à l'aide du code personnel à 6 chiffres.

Sécurité du pointage :

Le code de pointage est généré aléatoirement et hashé côté backend. L'appel à l'API de validation est protégé par un middleware administrateur pour éviter les tentatives de brute-force externes.

4. Architecture Technique
4
Architecture Technique

4.1 — Description de l'Architecture

L'application FITZONE repose sur une architecture découplée avec une séparation stricte des responsabilités (Client-Serveur). Cette approche garantit flexibilité, sécurité et fluidité dans les flux applicatifs.

Composants du socle technique :

  • Frontend (React Client) : Application Single Page (SPA) hautement interactive utilisant des routes protégées, des services Axios structurés et une gestion globale du statut d'authentification utilisateur via Context API.
  • Backend (Laravel REST API) : API robuste exposant des points d'accès sécurisés. La couche sécurité repose sur des jetons JWT cryptés qui identifient de manière unique le membre ou l'administrateur.
  • Base de Données (MySQL) : Stockage persistant hautement optimisé pour les jointures et indexé pour les requêtes rapides requises lors de la validation physique du code de pointage par les membres.

Conclusion du Dossier de Conception :

Ce dossier présente la structure logicielle, conceptuelle et visuelle complète développée pour le projet FITZONE de mi-avril à fin juin 2026. La conception assure une base solide, validée et documentée pour toutes les phases de maintenance futures et d'évolution fonctionnelle de l'application.
